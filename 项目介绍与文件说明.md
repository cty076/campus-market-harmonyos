# 校园二手好物项目介绍与文件说明

本文档用于说明 `code` 目录下 HarmonyOS ArkTS 项目的整体设计、功能链路、目录结构，以及每个主要文件的作用。可以作为课程设计报告、讲解视频脚本和答辩说明的参考材料。

## 1. 项目基本信息

项目目录：

```text
D:\mobile_app\final_work_wzq\code
```

项目名称：

```text
校园二手好物
```

技术栈：

```text
HarmonyOS
ArkTS
ArkUI
DevEco Studio
Hvigor
```

项目定位：

```text
本项目是一个面向校园学生的二手物品浏览应用，主要展示教材、考研资料、数码用品、宿舍生活用品和运动用品等校园二手商品。用户可以在首页浏览商品，切换分类，搜索关键词，进入商品详情页，并进行收藏或联系卖家的状态反馈。
```

## 2. 项目功能概述

本项目围绕一条完整功能链路设计：

```text
首页浏览商品 -> 分类切换 -> 搜索筛选 -> 查看商品详情 -> 收藏或联系卖家 -> 状态反馈
```

已实现的主要功能包括：

```text
1. 首页商品列表展示
2. 多分类 Tabs 切换
3. 搜索框关键词过滤
4. List + LazyForEach 商品懒加载
5. 下拉刷新状态显示
6. 列表到底提示
7. 商品详情页
8. 收藏状态切换
9. 我的收藏页
10. 联系卖家的页面内状态反馈
```

## 5. 核心源码文件说明

### 5.1 common 目录

#### `entry/src/main/ets/common/CommonConstants.ets`

作用：

```text
集中管理项目中常用的颜色、尺寸、间距、分类名称和加载数量等常量。
```

主要内容：

```text
1. 页面背景色、卡片背景色、主色、文字颜色等 UI 配色
2. 页面边距、卡片圆角、商品图片尺寸、搜索框高度等布局尺寸
3. 初始加载数量、每次加载数量、最大可见数量
4. 商品分类数组：全部、教材、考研、数码、生活、运动
```

为什么这样设计：

```text
把常量集中到一个文件后，页面和组件不用到处写重复数字和颜色。后续如果想统一调整界面风格，只需要改这个文件。
```

### 5.2 entryability 目录

#### `entry/src/main/ets/entryability/EntryAbility.ets`

作用：

```text
应用入口 Ability 文件，负责应用生命周期管理和加载首页。
```

主要内容：

```text
1. onCreate：应用创建时设置颜色模式并打印日志
2. onWindowStageCreate：窗口创建后加载 pages/Index 首页
3. onForeground / onBackground：记录应用前后台状态
4. onDestroy：应用销毁时记录日志
```

在项目中的意义：

```text
这是 HarmonyOS 应用启动时首先进入的能力类。它把应用主窗口和首页页面连接起来。
```

### 5.3 entrybackupability 目录

#### `entry/src/main/ets/entrybackupability/EntryBackupAbility.ets`

作用：

```text
应用备份扩展能力文件，由 DevEco 模板生成，用于支持系统备份相关扩展能力。
```

本项目中的使用情况：

```text
业务功能没有直接依赖它，保留它是为了保持 DevEco 默认工程结构完整。
```

## 6. 页面文件说明

页面文件位于：

```text
entry/src/main/ets/pages/
```

### 6.1 `Index.ets`

页面名称：

```text
首页
```

作用：

```text
应用主页面，展示项目标题、说明文字、搜索框、收藏入口和商品分类列表。
```

包含组件：

```text
SearchBarComponent
CampusTabBar
```

主要功能：

```text
1. 显示应用标题“校园二手好物”
2. 显示项目说明“浏览教材、考研资料、数码和宿舍生活用品”
3. 维护搜索关键词 keyword 状态
4. 点击“收藏”按钮进入 FavoritePage
5. 把 keyword 传给搜索框和分类列表组件
```

数据流：

```text
用户在搜索框输入关键词 -> Index.ets 中 keyword 改变 -> CampusTabBar 根据 keyword 更新商品列表
```

### 6.2 `DetailPage.ets`

页面名称：

```text
商品详情页
```

作用：

```text
展示某个商品的完整信息，并提供收藏和联系卖家的状态反馈。
```

主要功能：

```text
1. 从路由参数中读取 goodsId
2. 根据 goodsId 查询商品数据
3. 展示商品图片、标题、价格、分类、新旧程度、发布时间
4. 展示商品描述和卖家信息
5. 点击“收藏”或“取消收藏”切换收藏状态
6. 点击“联系卖家”显示页面内反馈信息
```

关键状态：

```text
goods            当前详情页展示的商品
isFavorite       当前商品是否已收藏
feedbackMessage  页面内状态反馈文案
```

页面跳转来源：

```text
GoodsCardComponent 点击后跳转到 DetailPage，并传入 goodsId。
```

### 6.3 `FavoritePage.ets`

页面名称：

```text
我的收藏页
```

作用：

```text
展示用户已经收藏的商品列表。
```

主要功能：

```text
1. 页面出现时读取 FavoriteStore 中保存的收藏商品 id
2. 根据 id 找到对应商品对象
3. 有收藏时展示商品卡片列表
4. 没有收藏时展示 EmptyView 空状态
```

与其他文件关系：

```text
FavoritePage -> FavoriteStore -> CampusGoodsData -> GoodsCardComponent
```

## 7. 组件文件说明

组件文件位于：

```text
entry/src/main/ets/view/
```

### 7.1 `SearchBarComponent.ets`

作用：

```text
封装首页搜索框。
```

主要功能：

```text
1. 展示搜索提示“搜索教材、考研资料、宿舍用品”
2. 通过 @Link keyword 和首页共享关键词状态
3. 输入变化时更新 keyword
```

为什么单独拆分：

```text
搜索框是一个独立交互模块，单独拆分后首页代码更简洁，报告中也能单独说明搜索功能。
```

### 7.2 `CampusTabBar.ets`

作用：

```text
封装商品分类 Tabs、下拉刷新状态和分类数据切换逻辑。
```

主要功能：

```text
1. 使用 Tabs 显示分类：全部、教材、考研、数码、生活、运动
2. 监听当前分类 currentIndex
3. 根据分类和搜索关键词更新 CampusGoodsDataSource
4. 通过 TouchEvent 判断下拉距离并显示 PullRefreshLayout
5. 把数据源传给 GoodsListComponent
```

关键状态：

```text
keyword          首页传入的搜索关键词
currentIndex     当前选中的分类下标
currentCategory  当前选中的分类名称
refreshStatus    是否显示刷新状态
dataSource       当前列表数据源
```

与参考示例的关系：

```text
该组件参考了商品列表示例中的 Tabs 分类切换和下拉刷新思路，但分类和业务数据已经改成校园二手场景。
```

### 7.3 `GoodsListComponent.ets`

作用：

```text
封装商品列表。
```

主要功能：

```text
1. 使用 List 展示商品
2. 使用 LazyForEach 按需渲染商品卡片
3. 列表为空时展示 EmptyView
4. 列表到底时触发 onReachEnd 加载更多
5. 列表底部显示 BottomTip
```

核心技术点：

```text
LazyForEach + IDataSource
```

为什么重要：

```text
这是本项目对应课程要求中“列表滑动、懒加载、到底提示”的核心组件。
```

### 7.4 `GoodsCardComponent.ets`

作用：

```text
封装单个商品卡片。
```

主要功能：

```text
1. 展示商品图片
2. 展示商品标题
3. 展示商品描述摘要
4. 展示价格、新旧程度、发布时间
5. 显示分类标签或已收藏状态
6. 点击卡片跳转到 DetailPage
```

路由行为：

```text
点击商品卡片 -> pushUrl 到 pages/DetailPage -> 传入 goodsId
```

### 7.5 `PullRefreshLayout.ets`

作用：

```text
封装下拉刷新时显示的提示布局。
```

主要内容：

```text
1. LoadingProgress 加载动画
2. “正在刷新校园好物”文字提示
3. 绿色边框和浅色背景提示刷新状态
```

对应课程要求：

```text
下拉刷新功能
```

### 7.6 `BottomTip.ets`

作用：

```text
封装列表底部提示。
```

主要逻辑：

```text
1. 如果还有更多数据，显示“继续上滑加载更多”
2. 如果没有更多数据，显示“已经到底了”
```

对应课程要求：

```text
到底提示
```

### 7.7 `EmptyView.ets`

作用：

```text
封装空状态视图。
```

使用场景：

```text
1. 搜索没有匹配商品时
2. 我的收藏为空时
```

好处：

```text
增强交互完整度，避免页面空白。
```

## 8. 数据层文件说明

数据层文件位于：

```text
entry/src/main/ets/viewmodel/
```

### 8.1 `GoodsModel.ets`

作用：

```text
定义商品数据模型。
```

字段说明：

```text
id           商品唯一标识
title        商品标题
category     商品分类
price        商品价格
condition    新旧程度
description  商品描述
seller       卖家信息
publishTime  发布时间
image        商品图片资源
tags         搜索标签
```

方法：

```text
matches(keyword, category)
```

该方法用于判断商品是否符合当前搜索关键词和分类条件。

### 8.2 `CampusGoodsData.ets`

作用：

```text
提供校园二手商品模拟数据。
```

包含商品示例：

```text
1. 高等数学教材上下册
2. 英语四六级真题套卷
3. 考研政治冲刺资料
4. 蓝牙降噪耳机
5. 宿舍护眼台灯
6. 室外篮球
7. 机械键盘 87 键
8. 书桌收纳盒套装
9. 计算机网络教材
10. 折叠小桌板
11. 考研数学讲义
12. 羽毛球拍双拍
```

导出函数：

```text
findGoodsById(id)
filterGoods(keyword, category)
```

作用分别是：

```text
findGoodsById       根据 id 查找商品，详情页使用
filterGoods         根据关键词和分类筛选商品，首页列表使用
```

### 8.3 `CampusGoodsDataSource.ets`

作用：

```text
为 LazyForEach 提供数据源。
```

核心能力：

```text
1. 实现 IDataSource 接口
2. 管理 DataChangeListener
3. totalCount 返回当前可见数据数量
4. getData 根据下标返回商品
5. updateFilter 根据关键词和分类刷新数据
6. refresh 重置当前列表
7. loadMore 加载更多数据
8. hasMore 判断是否还有更多数据
9. isEmpty 判断当前列表是否为空
```

对应课程要求：

```text
LazyForEach 懒加载
上拉加载更多
列表状态更新
```

### 8.4 `FavoriteStore.ets`

作用：

```text
管理收藏商品 id。
```

主要方法：

```text
isFavorite(goodsId)  判断商品是否已收藏
toggle(goodsId)      收藏或取消收藏
remove(goodsId)      删除收藏
getIds()             获取全部收藏 id
```

说明：

```text
当前收藏状态保存在内存中，适合课程作业演示。应用重启后收藏状态不会持久保存。如果后续扩展，可以接入 Preferences 做本地持久化。
```

## 13. 功能实现逻辑梳理

### 13.1 首页浏览流程

```text
EntryAbility
  -> 加载 pages/Index
  -> Index 显示标题、搜索框和 CampusTabBar
  -> CampusTabBar 创建 CampusGoodsDataSource
  -> GoodsListComponent 使用 LazyForEach 展示商品
  -> GoodsCardComponent 展示每个商品卡片
```

### 13.2 搜索筛选流程

```text
用户输入关键词
  -> SearchBarComponent 修改 keyword
  -> Index 中 keyword 状态更新
  -> CampusTabBar 监听 keyword 变化
  -> CampusGoodsDataSource.updateFilter(keyword, category)
  -> filterGoods 重新筛选商品
  -> LazyForEach 刷新列表
```

### 13.3 分类切换流程

```text
用户点击分类 Tabs
  -> CampusTabBar 更新 currentIndex 和 currentCategory
  -> CampusGoodsDataSource.updateFilter(keyword, category)
  -> 列表刷新为对应分类商品
```

### 13.4 下拉刷新流程

```text
用户下拉页面
  -> CampusTabBar 监听 TouchEvent
  -> 下拉距离超过 REFRESH_DISTANCE
  -> refreshStatus 变为 true
  -> 显示 PullRefreshLayout
  -> dataSource.refresh()
  -> 短暂延迟后隐藏刷新提示
```

### 13.5 上拉加载与到底提示流程

```text
用户滑动到列表底部
  -> GoodsListComponent.onReachEnd
  -> 调用 dataSource.loadMore()
  -> 可见商品数量增加
  -> 如果还有数据，BottomTip 显示“继续上滑加载更多”
  -> 如果没有数据，BottomTip 显示“已经到底了”
```

### 13.6 商品详情流程

```text
用户点击商品卡片
  -> GoodsCardComponent pushUrl 到 pages/DetailPage
  -> 路由参数传入 goodsId
  -> DetailPage 根据 goodsId 调用 findGoodsById
  -> 展示商品完整信息
```

### 13.7 收藏流程

```text
用户在详情页点击收藏
  -> DetailPage 调用 FavoriteStore.toggle(goodsId)
  -> isFavorite 状态改变
  -> 页面显示“已加入收藏”或“已取消收藏”
  -> 用户回到首页点击收藏入口
  -> FavoritePage 从 FavoriteStore 读取收藏 id
  -> 展示收藏商品列表
```
