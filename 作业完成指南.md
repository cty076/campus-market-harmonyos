# 移动应用开发课程大作业完成指南

本文档用于说明本次大作业的完成思路、工程框架、报告结构、视频准备方式，以及后续继续开发时的执行顺序。当前项目根目录为：

```text
D:\mobile_app\final_work_wzq
```

## 1. 作业要求概括

根据 `《移动应用开发》课程大作业v2026 (1).pdf`，最终提交文件名应为：

```text
学号-姓名.zip
```

压缩包内需要包含：

```text
1. 鸿蒙源代码工程
2. 课程设计报告 Word 文档，不少于 2000 字
3. 配套讲解视频 MP4，不超过 50M
```

截止时间：

```text
第 17 周周五 17:00
```

作业重点不是堆砌一个完整 APP，而是做透一条完整功能链路，例如：

```text
内容浏览 -> 筛选/搜索 -> 详情 -> 状态反馈
```

PDF 中提供的“商城”和“信息流”示例只能作为参考，不能直接提交原示例代码。

## 3. 推荐正式选题

正式项目建议做：

```text
校园二手好物
```

业务定位：

```text
面向校园学生的二手书籍和生活用品浏览应用。
```

核心功能链路：

```text
首页浏览商品 -> 分类切换 -> 搜索筛选 -> 查看详情 -> 收藏/联系卖家 -> 状态反馈
```

这样做的原因：

```text
1. 与商品列表示例有技术关联，方便参考实现
2. 业务内容是自己的，不属于直接照搬示例
3. 可以自然覆盖列表、分类、搜索、详情、刷新、加载、状态反馈
4. 报告容易写清楚需求分析、设计思路和实现方案
```
